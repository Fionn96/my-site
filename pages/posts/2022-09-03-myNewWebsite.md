## 起因

### 笔记软件的劝退

因为学习的过程中记录了一些笔记，需要存储在一个地方以便查阅。B 站那时经常给我推各种笔记软件的使用技巧，尤其是印象笔记的视频非常多，所以在很长一段时间我都在使用印象笔记。直到被它的祖传 bug 以及层出不绝的 vip、svip、svvip 所劝退（我还真的充了3年，然而它还是隔三岔五提醒我续费，可能是需要再续 500 年）。语雀、为知笔记也都试过，但总感觉不是那么得劲儿。Notion 我倒是很喜欢，可因为国内糟糕的访问速度和英文界面最后还是被劝退了（很多视频教程都是基于英文界面来做的，对于英文不好的蠢萌真的挺需要下定决心去学习的）。

在见证一个个笔记软件一步步变成一个个氪金软件（对没错，说的就是印象笔记，它就差增加贷款功能了），我觉得我应该寻找其他新的解决方案了。一方面我需要随时随地进行查阅（有时候个人电脑不在身边）；另一方面，自己还有一丁点儿的分享欲，所以还得找一个既可以本地化储存又可以网络备份同步的方案。

我尝试过坚果云、OneDrive 等同步方案，但没有找到满意的。

这个时候，我想到去做一个博客网站。

### 原来博客是可以生成的

在 B 站和 CSDN 搜了一圈，发现市面上有很多教学视频，教你怎么以项目的方式手写一个博客。有前端、有后台、有数据库等。有完整的各个功能：文本编辑器、标签、时间轴、评论系统等。我大致看了看，直接选择了放弃搭建博客——我就想找个地方记东西咋这么麻烦？博主们一定都是世界上最勤快的人。但事实证明：只有懒惰才会丰富人的知识。

我将注意力转移到了各个可以写文章的平台：B 站、知乎、公众号等等，不过还是没有找到满意的方案，不是广告满天飞，就是 UI 超越了我的审美，要么就是不登陆看不了全文等骚操作，很快我就放弃了。

再后来因为神奇的原因，我了解到了**静态网站生成器**。

## Github Pages

### Jekyll

后来在~~同性交友社区~~ Github 上发现有个东西叫 Github Pages，把前端页面及静态资源丢在上面就可以直接通过 Github 的域名访问，简直不要太酷好不好！

于是我照着别人的教程操作了一遍，并成功通过带有自己名字的域名访问后，差点打算硬撸 html 写博客，还好我读完了文档（教程中只介绍了可以放 html）。ps：打算使用某个技术，通读文档真的是好习惯。

我得知 Github Pages 不但可以放置 html，还可以放置以 `.md` 结尾的 markdown 文件。因为 Github Pages 默认集成了一个叫做 Jekyll 的静态网站生成器。所以写博客原来是可以不用写代码的！这简直打开了我新世界的大门！

那段时间，我都在使用这种方式记笔记，享受着 Github Pages 自带 Jekyll 带来的便利，因为真的很方便，把 markdown 文件上传到 Github 上，过一小会儿就可以在网站上看到了。直到后来逛了不少别人的博客，眼光变高了，我开始觉得它丑得有点影响阅读体验😂。

当然丑是可以通过使用主题解决的，我在 Github 上找到了一个名为 [Gungnir](https://github.com/Renovamen/jekyll-theme-gungnir)的主题。

### Gungnir

Gungnir 是一个基于 Jekyll 的主题，它的设计满足了我最主要的需求：

- 有一个即使没什么丰富经历也可以撑起来的主页（通过二次元大图hhh）。
- 它同时拥有文档和博客功能，文档是文档，博客是博客，太 nice 了。
- 美观（有一说一是真的好看）。

我花了大量的时间去修改和美化主题，甚至开始学习 JavaScript 和 CSS ——

是的，我花了非常多的时间去修改维护博客主题样式，以至于我几乎要忘记了初衷只是找个地方放笔记......等回过神来，想要回归简单的设计，但是代码已经被我改的乱七八糟了根本无从下手，于是乎基于又菜又爱玩的心理，我转向了另一个新网站。

## Vitesse

这个项目是在 GitHub 上发现的，这个由 antfu 编写的应用模板强大且优雅，在看到 antfu 使用 Vitesse 制作的个人主页后，瞬间就被它圈粉了，所以我选择在 Vitesse 上搭建我的新个人网站，它可以在很短的时间内搞定一个美观、轻量、快速的带有博客页面的个人网站。

无论如何，这个网站仍然是 WIP, 希望这是我最后一个个人网站了。
